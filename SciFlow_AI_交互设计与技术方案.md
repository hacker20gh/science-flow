### 1.3 知识面板（Brain）— 课题的"大脑"

#### 它是什么？

知识面板是**课题所有已知信息的实时汇总视图**。

打个比方：
- **时间线** = 你的实验日记（按时间顺序记录"发生了什么"）
- **知识面板** = 你的课题大脑（把所有信息消化后，提炼出"我现在知道什么"）

时间线越长，知识面板越丰富。每加一篇文献、每完成一组实验，知识面板自动更新。

#### 它包含什么？

知识面板由 **3 个模块** 组成，从上到下依次是：

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │

│  │  模块 1：机制矩阵（核心表格）                                  │   │
│  │  "这个领域里，哪些通路怎么变，谁跟谁一致，谁跟谁矛盾"            │   │

#### 模块 1：机制矩阵

**一句话**：把多篇文献 + 你的实验结果，浓缩成一张对比表格。

**它解决的问题**：正常情况下，你要读完 10 篇论文、记住每篇的关键发现、在脑子里做对比——这非常累。机制矩阵帮你把这件事可视化了。

```
┌─ 机制矩阵 ──────────────────────────────────────────────────────────────┐
│                                                                         │
│  筛选：[全部通路 ▾] [全部细胞系 ▾] [年份 ▾] [显示冲突 ✓] [显示空白 ✓]     │
│                                                                         │
│              │ NF-κB  │ PD-L1  │ Apoptosis │ Proliferation │ Migration  │
│  ────────────┼────────┼────────┼───────────┼───────────────┼────────────│
│  Liu 2024    │  ↑**   │  ↑*    │   ↑*      │     ↓*        │    —       │
│  (IF=15.3)   │        │        │           │               │            │
│  ────────────┼────────┼────────┼───────────┼───────────────┼────────────│
│  Chen 2023   │  ↑*    │  ↓*  ⚠️│   ↑**     │     —         │    ↓*      │
│  (IF=8.7)    │        │        │           │               │            │
│  ────────────┼────────┼────────┼───────────┼───────────────┼────────────│
│  Zhang 2022  │  —     │  ↑*    │   —       │     ↓**       │    —       │
│  (IF=12.1)   │        │        │           │               │            │
│  ────────────┼────────┼────────┼───────────┼───────────────┼────────────│
│  🧪 Exp#2    │  ↑*    │  ↑*    │   ↑       │     ↓         │    —       │
│  (你的实验)   │        │        │           │               │            │
│  ────────────┼────────┼────────┼───────────┼───────────────┼────────────│
│  📊 一致性    │ 100%   │  67%⚠️ │  100%     │    100%       │   100%     │
│                                                                         │
│  ⚠️ PD-L1 列存在冲突：Chen 2023 报道下调，其余报道上调                     │
│  🔍 空白：Migration 在 HCC 模型中研究不足（仅 1/4 文献涉及）                │
│                                                                         │
│  [添加列（通路/表型）] [导出 CSV] [导出 LaTeX] [添加你的实验数据]             │
└─────────────────────────────────────────────────────────────────────────┘
```

**怎么读这张表**：
- **行** = 每篇文献（或你的实验），一行一个来源
- **列** = 你关心的维度（信号通路、表型、指标等）
- **单元格** = 这篇文献在这个维度上的发现
  - `↑` = 上调/增强
  - `↓` = 下调/抑制
  - `—` = 没研究这个
  - `*` = p<0.05，`**` = p<0.01（证据强度）
- **最后一行** = 一致性评分，告诉你这个维度上文献是否达成共识

**颜色编码**（一眼看出重点）：

```
🟢 绿色背景   = 所有文献一致（共识区）
🟠 橙色背景   = 文献之间有矛盾（争议区）  ← 最值得关注
⚪ 浅灰背景   = 空白，没人研究过（Gap）    ← 潜在创新点
🔵 蓝色边框   = 你自己的实验数据           ← 区分于文献
```

**交互规则**：

| 操作 | 效果 |
|------|------|
| **悬停单元格** | 弹出 Tooltip：药物浓度、处理时间、细胞系、原文片段 |
| **点击单元格** | 展开详情面板：原文引用 + 论文段落 + "查看原文"链接 |
| **点击 ⚠️ 冲突标记** | 展开冲突分析：哪些文献矛盾、可能原因（浓度/时间/细胞系差异） |
| **点击空白单元格** | 提示："这个组合没人研究过，要设计实验验证吗？" → 跳转实验设计 |
| **添加行** | 把你的实验数据加入矩阵，跟文献做对比 |
| **添加列** | 搜索文献中提到但未纳入的通路/表型，或手动输入 |
| **导出** | CSV（数据）、LaTeX table（论文）、PNG（汇报用） |

**一句话**：你当前在验证什么假设，证据积累到什么程度了。

```
┌─ 假设追踪器 ────────────────────────────────────────────────────────────┐
│                                                                         │
│  ┌─ 假设 #1（当前活跃）──────────────────────────────────────────────┐  │
│  │                                                                   │  │
│  │  "sorafenib 通过 NF-κB 通路上调 HCC 细胞中的 PD-L1 表达"           │  │
│  │                                                                   │  │
│  │  状态：🔄 验证中                                                   │  │
│  │                                                                   │  │
│  │  支持证据：                                                        │  │
│  │  ✅ Liu 2024：NF-κB 与 PD-L1 正相关（IF=15.3）                     │  │
│  │  ✅ Exp#2：sorafenib 2-3μM 上调 PD-L1（你的数据）                  │  │
│  │  ✅ Exp#3：NF-κB 抑制剂减弱了 PD-L1 上调（你的数据）               │  │
│  │                                                                   │  │
│  │  反对证据：                                                        │  │
│  │  ⚠️ Chen 2023：10μM sorafenib 下调 PD-L1（但浓度不同）             │  │
│  │                                                                   │  │

│  │                                                                   │  │
│  │  [标记为已验证 ✅]  [修订假设]  [设计补充实验]                       │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  ┌─ 假设 #2（已修订）────────────────────────────────────────────────┐  │
│  │  "sorafenib 在所有浓度下都上调 PD-L1"                              │  │
│  │  状态：❌ 已修订 → 基于 Exp#1 失败（5μM 细胞毒性）修订为假设 #1      │  │
│  └───────────────────────────────────────────────────────────────────┘  │
│                                                                         │
│  [+ 提出新假设]                                                         │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**为什么需要它**：
- 做实验容易迷失方向——"我到底在验证什么来着？"
- 假设不是一成不变的，实验结果可能推翻或修订假设
- 追踪器记录了假设的演变过程，写论文时很有用

#### 模块 3：待办清单

**一句话**：你的课题还缺什么。

```
┌─ 待办清单 ──────────────────────────────────────────────────────────────┐
│                                                                         │
│  🔴 必须完成（缺少会导致结论不成立）                                       │
│  ☐ Exp#3 缺少阳性对照（TNF-α）                                          │
│     → 系统建议：添加 TNF-α 作为 NF-κB 激活的阳性参照                     │
│     [一键补全到 Protocol]                                                │
│  ☐ 未做溶剂毒性基线检测                                                  │
│     → 系统建议：单独检测 DMSO 对 Huh7 细胞活力的影响                      │
│     [一键补全到 Protocol]                                                │
│                                                                         │
│  🟡 建议完成（会让论文更强）                                              │
│  ☐ 多重比较校正（你测了 4 个指标，需要 Bonferroni 或 FDR）                │
│     → 系统建议：在统计分析中添加 FDR 校正                                 │
│     [📚 了解多重比较校正]                                                 │
│  ☐ 增加生物学重复（当前 n=3，建议 n=5 以提高功效）                        │
│     → 系统建议：补充 2 组独立实验                                         │
│     [📚 了解统计功效]                                                    │
│                                                                         │
│  🟢 已完成                                                               │
│  ✅ Vehicle 对照完整                                                     │
│  ✅ 生物学重复 ≥ 3                                                       │
│  ✅ Protocol 详细记录                                                    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**为什么需要它**：
- 写论文或被导师 review 时才发现"忘了做对照"——代价很大
- 系统基于你的实验设计自动检查，提前发现问题
- 每个待办项都可以一键操作（补到 Protocol、跳去学习、标记完成）

#### 三个模块之间的关系

```
  文献搜索                    实验完成                    数据上传
     │                          │                          │
     ▼                          ▼                          ▼
┌──────────┐              ┌──────────┐              ┌──────────┐
│ 机制矩阵  │ ──更新──→    │ 假设追踪  │ ──更新──→    │ 待办清单  │
│ （知识积累）│              │ （方向判断）│              │ （行动指南）│
└──────────┘              └──────────┘              └──────────┘

具体来说：

1. 你搜了 10 篇文献 → 机制矩阵多了 10 行
2. 矩阵显示 NF-κB 和 PD-L1 有强关联 → 假设追踪器建议新假设
3. 你设计实验验证假设 → 待办清单检查你的 Protocol 是否完整
4. 实验完成，数据上传 → 机制矩阵多了一行（你的数据），假设追踪器更新证据强度
5. 待办清单发现缺阳性对照 → 提醒你补实验

三个模块联动，始终保持课题状态的最新视图。
```

|------|------|

##### 过程助手的触发规则

不是每次都弹，而是在**关键决策点**才介入：

| 触发条件 | 助手行为 | 介入方式 |
|---------|---------|---------|
| 样本量 < 建议值 | 计算统计功效，推荐样本量 | 自动弹出卡片 |
| 缺少必要对照组 | 列出缺失对照 + 文献参考 | 自动弹出卡片 |
| 浓度超出文献安全范围 | 对比文献 + 历史实验数据 | 自动弹出卡片 ⚠️ |
| 统计方法选择 | 根据数据特征推荐方法 | 自动弹出卡片 |
| 结果与文献矛盾 | 分析可能原因 | 用户主动询问时 |
| 写作每个章节 | 结构建议 + 引用推荐 | 自动弹出卡片 |
| 用户停留在某个步骤 >30 秒 | 可能需要帮助 | 轻提示（可关闭） |

**用户可以设置助手介入程度**：

```
过程助手设置：
  ○ 全程指导 — 每个决策点都介入（适合新手）
  ● 关键提醒 — 只在高风险决策时介入（推荐）
  ○ 静默模式 — 只在我主动问的时候回答（适合资深用户）
```

#### 与 AI 助手的联动

AI 助手在对话中自然地融入教学：

```
用户：我的 p 值是 0.06，是不是就没效果了？

AI：不一定。p=0.06 意味着在 5% 显著性水平下未达到统计显著性，
但这不代表"没有效果"。有几个角度需要考虑：

1. 效应量（Effect Size）：你的 fold change 是多少？
   如果效应量大但 p 稍高，可能是样本量不够。

2. 统计功效：你的 n=3 可能功效不足。
   我帮你算了一下，以你的效应量，n=5 才能达到 80% 功效。

3. 从生物学角度：0.06 仍然提供了"倾向性证据"，
   值得扩大样本量再验证。

想深入了解？→ [打开"什么是 p 值"知识卡片]
想优化实验？→ [打开统计功效计算器，算算需要多少样本]

// lib/llm/prompts/extraction.ts

export const EXTRACTION_PROMPT = `You are a biomedical literature analysis expert.
Extract structured information from the following paper.

For each experimental finding, extract:
1. drug_intervention: { name, concentration, duration }
2. model_system: { cell_line, species, passage_range }
3. pathway_change: { pathway_name, direction: "up"|"down"|"no_change", significance: "p<0.05"|"p<0.01"|"p<0.001"|"ns", method }
4. phenotype_change: { phenotype, direction, significance, method }

#### 防幻觉策略

科学场景对准确性要求极高，必须有多层防幻觉机制：

| 层级 | 策略 | 说明 |
|------|------|------|
| **Prompt 层** | 要求附带原文引用 | 每个提取结果必须附带原文片段，用户可溯源验证 |
| **输出层** | Structured Output (JSON Schema) | Claude 的 tool use 强制输出符合 schema 的 JSON，减少格式错误 |
| **验证层** | 置信度评分 | LLM 对每个提取给出 0-1 置信度，低于 0.7 的标记为"待验证" |
| **用户层** | 可编辑 + 可验证 | 用户可以修正任何提取结果，标记为"已验证" |
| **交叉验证** | 多文献一致性检查 | 同一结论在多篇文献中出现 → 置信度提升 |

### 2.9 多平台策略：Web + App

用户需求：后期需要同时支持网页端和移动端 App。

#### 方案对比

| 方案 | 代码复用 | 开发成本 | 体验 | 推荐阶段 |
|------|---------|---------|------|---------|
| **A. Next.js 响应式 + PWA** | 100% 同一套代码 | 最低 | Web 体验优秀，移动端可用但非原生 | MVP 阶段 ✅ |
| **B. Next.js (Web) + React Native (App)** | 业务逻辑共享 ~60%，UI 各写 | 中等 | 两端都是原生体验 | 增长阶段 |
| **C. Next.js (Web) + Capacitor (App 壳)** | ~95% 复用 | 低 | 原生壳装 Web，体验介于 PWA 和原生之间 | 过渡方案 |

#### 推荐路径：PWA 优先 → React Native 后补

```
MVP 阶段（现在）                增长阶段（用户量 >1000）
┌─────────────────┐           ┌─────────────────────┐
│  Next.js 响应式   │           │  Next.js（Web 端）    │
│  + PWA 支持      │    →     │  +                   │
│                  │           │  React Native（App）  │
│  一套代码覆盖：    │           │                      │
│  • 桌面浏览器     │           │  共享：                │
│  • 手机浏览器     │           │  • API 层             │
│  • 可安装到桌面   │           │  • 业务逻辑            │
│  • 离线缓存       │           │  • 数据模型            │
│                  │           │  • LLM prompts        │
└─────────────────┘           └─────────────────────┘
```

#### 为什么 MVP 先做 PWA

1. **一套代码**：不需要维护两套代码库，一个人也能搞定
2. **移动端可用**：响应式设计 + PWA = 手机浏览器能用，可添加到主屏幕
3. **科研场景特点**：核心操作（文献搜索、矩阵分析、论文写作）在电脑上做，手机上主要是查看进度、接收通知
4. **零额外成本**：Next.js 通过 `next-pwa` 插件即可支持 PWA
5. **快速验证**：先验证产品价值，再投入做原生 App

#### MVP 阶段的移动端适配

```
手机端不做什么（重操作留给桌面）：
  ✗ 机制矩阵的复杂交互（列管理、批量操作）
  ✗ 论文写作和编辑
  ✗ 数据上传和分析

手机端做什么（轻操作 + 查看）：
  ✅ 查看项目时间线
  ✅ 查看机制矩阵（只读，可缩放）
  ✅ 接收实验状态通知
  ✅ AI 助手对话（快速提问）
  ✅ 记录实验备注/拍照（实验室场景）
  ✅ 查看 Protocol（实验时对照）
```

#### 响应式布局适配

```
桌面端（>1024px）：三栏布局
┌────────┬──────────────────────┬──────────┐
│ 侧边栏  │      主内容区         │ AI 助手   │
└────────┴──────────────────────┴──────────┘

平板端（768-1024px）：两栏布局
┌────────┬──────────────────────────────────┐
│ 侧边栏  │          主内容区                │
│（可收起）│     AI 助手变为浮窗/底部抽屉      │
└────────┴──────────────────────────────────┘

手机端（<768px）：单栏布局
┌──────────────────────────────────────────┐
│ [☰]  课题名                    [🤖 AI]   │
├──────────────────────────────────────────┤
│                                          │
│           主内容区（全屏）                  │
│      Tab 切换：时间线/知识/草稿             │
│                                          │
├──────────────────────────────────────────┤
│ [时间线] [知识] [实验] [论文] [更多]        │
└──────────────────────────────────────────┘
```

#### 后期 App 端技术方案（React Native）

当用户量增长、需要原生 App 时：

| 层 | Web 端 | App 端 | 共享策略 |
|---|--------|--------|---------|
| **UI** | Next.js + shadcn/ui | React Native + NativeWind | 不共享，各自原生 |
| **业务逻辑** | TypeScript | TypeScript | 通过 monorepo 共享 `packages/shared/` |
| **API 调用** | fetch / TanStack Query | fetch / TanStack Query | 共享 API client |
| **数据模型** | Prisma schema → TS types | 同左 | 共享 types |
| **LLM Prompts** | 字符串模板 | 同左 | 共享 `packages/prompts/` |
| **后端** | Next.js API Routes | 同一个后端 | 共享 |

Monorepo 结构：

```
sciflow/
├── apps/
│   ├── web/              # Next.js Web 端
│   └── mobile/           # React Native App 端
├── packages/
│   ├── shared/           # 共享业务逻辑、类型、工具函数
│   ├── api-client/       # 统一 API 调用层
│   └── prompts/          # LLM Prompt 模板
└── package.json          # Turborepo / pnpm workspace
```

#### App 端独有功能（利用移动端特性）

| 功能 | 说明 | 利用的移动端能力 |
|------|------|---------------|
| **实验拍照记录** | 拍 Western blot 条带、细胞照片，自动关联实验 | 相机 + 图片上传 |
| **扫码添加文献** | 扫描论文 PDF 上的 DOI 二维码，自动导入 | 相机 + 二维码识别 |

| **推送通知** | "你的数据分析完成了"、"有新文献与你的假设相关" | 推送通知 |
| **语音备注** | 实验中语音记录观察，自动转文字关联到时间线 | 语音识别 |
